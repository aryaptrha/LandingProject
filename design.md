# Design Guidelines
**Project Style:** 2D Pixel × Glass UI × Pastel Theme

---

# Design Philosophy

The interface should feel like a handcrafted indie game rather than an AI-generated landing page.

The visual language combines:

- Pixel-art aesthetics
- Flat pastel colors
- Frosted glass UI
- Clean spacing
- Minimal decoration

The design should feel:

- Calm
- Cozy
- Friendly
- Modern
- Intentional

Avoid making the interface look futuristic, cyberpunk, or generated by AI design tools.

---

# Core Principles

## 1. Pixel First

Every illustration should follow pixel art principles.

- 16x16
- 24x24
- 32x32
- 48x48
- 64x64

Use crisp edges.

Never blur pixel assets.

No anti-aliasing on illustrations.

---

## 2. Flat Colors Only

Use solid colors.

DO NOT use:

- gradients
- mesh gradients
- aurora backgrounds
- glowing blobs
- colorful lighting
- AI-style smooth transitions

Every surface should have a single color.

---

## 3. Glass UI

Glass should be subtle.

Recommended values:

Background Blur:
```
blur(12px)
```

Opacity:
```
75%–85%
```

Border:
```
1px solid rgba(255,255,255,.25)
```

Shadow:
```
0 8px 24px rgba(0,0,0,.08)
```

Avoid:

- heavy glow
- neon borders
- glassmorphism overload

Glass exists only for cards, dialogs, menus, and navigation.

# Typography

## Primary Font

Use rounded sans-serif fonts.

Recommended:

- Nunito
- M PLUS Rounded
- Fredoka
- Quicksand

Avoid:

- Serif
- Condensed
- Ultra-bold display fonts

---

## Pixel Font

Only use for:

- Logo
- Game title
- Small badges
- Achievement labels

Recommended:

- Pixelify Sans
- Press Start 2P
- Tiny5

Never use pixel fonts for body paragraphs.

---

# Spacing

Use an 8px spacing system.

```
4px
8px
16px
24px
32px
40px
48px
64px
```

Generous whitespace is preferred.

---

# Border Radius

Large components:

```
20px
```

Buttons:

```
16px
```

Input:

```
14px
```

Badges:

```
999px
```

Pixel illustrations should remain square.

---

# Buttons

Primary

- Solid pastel fill
- Dark text
- Small shadow
- Rounded corners

Hover

- Slightly darker fill
- Lift by 2px

Active

- Remove lift
- Slight shadow reduction

Avoid:

- glowing buttons
- animated gradients

---

# Cards

Cards should use glass.

Structure:

- Frosted background
- Thin white border
- Soft shadow
- Rounded corners

Padding:

```
24px
```

Gap:

```
16px
```

---

# Icons

Use pixel icons.

Recommended size:

```
24px
```

or

```
32px
```

Avoid:

- Material icons
- 3D icons
- Gradient icons
- Clay icons

---

# Illustrations

Illustrations should resemble handcrafted indie game assets.

Preferred subjects:

- trees
- flowers
- clouds
- stars
- books
- coffee
- desktop PC
- keyboard
- cats
- plants
- mailboxes
- tiny houses

Keep:

- simple
- low detail
- consistent pixel scale

---

# Background

Background should remain minimal.

Allowed:

- tiny pixel stars
- pixel clouds
- sparse decorative leaves
- dotted patterns
- subtle checker textures

Avoid:

- abstract blobs
- giant circles
- mesh backgrounds
- radial gradients

---

# Shadows

Soft only.

Examples:

```
0 6px 16px rgba(0,0,0,.06)

0 8px 24px rgba(0,0,0,.08)
```

Avoid:

```
0 20px 80px
```

or dramatic floating effects.

---

# Motion

Animations should feel lightweight.

Recommended duration:

```
150ms
```

or

```
200ms
```

Use:

- fade
- scale (1.00 → 1.02)
- slight lift
- opacity transition

Avoid:

- bouncing
- elastic effects
- exaggerated parallax
- spinning animations

---

Preferred max width:

```
1200px
```

Content width:

```
960px–1100px
```

Large breathing space around sections.

# Accessibility

Minimum text contrast:

```
4.5:1
```

Minimum clickable area:

```
44x44px
```

Visible keyboard focus is required.

Do not rely solely on color to indicate state.

---

# What to Avoid

## AI-Looking Design

Avoid:

- purple-blue gradients
- floating neon blobs
- mesh gradients
- excessive glow
- oversized hero illustrations
- random abstract shapes
- glass everywhere
- futuristic interfaces
- overly smooth organic curves

---

## Visual Noise

Avoid:

- too many colors
- excessive shadows
- multiple illustration styles
- inconsistent icon styles
- decorative clutter

---

## Pixel Art Mistakes

Never:

- stretch pixel art
- blur pixel assets
- anti-alias pixel artwork
- mix pixel art with realistic illustrations
- use different pixel scales

# Overall Goal

The website should look like it was designed by a skilled UI designer with a love for indie pixel games, not by an AI image generator. Every component should feel handcrafted, restrained, and cohesive, with flat pastel colors, crisp pixel artwork, and subtle glass effects that enhance usability rather than dominate the visual style.
